6 Class built with the purpose to manage configuration options or settings easily, there are two types of configs: private and public and they have different behaviors.
7 
8 Overview
9 ---
10 The class has a specific lifecycle:
11 
12 1. One or more instances are created:
13  - if no config file is passed to the constructor the file mh_config.json will be used to get and save configs.
14 '''php
15 <pre>
16 $config = new \MH\MH_Config();
17 </pre>
18  - if the FULL PATH of a config file is passed that file will be used to get and save configs.
19 '''php
20 <pre>
21 $config = new \MH\MH_Config(\__DIR__."/config.json");
22 </pre>
23 2. Value are retrieved or set (with magic setter and getter):
24  * Set:
25  - there is no limitation to set/overwrite public config
26 '''php
27 <pre>
28 $config->myPublicElement = "foo";
29 </pre>
30  - to set/overwrite a private item the override flag must be set to true otherwise the set will be ignored
31 '''php
32 <pre>
33 //ignored
34 $config->_myPrivateElement = "bar";
35 //set as aspected
36 $config->override = true;
37 $config->_myPrivateElement = "bar";
38 </pre>
39  * Get:
40  - if the desidered item starts with an underscore the getter will look into the private array and return the value, if the item is not found, false is returned.
41 '''php
42 <pre>
43 echo $config->_myPrivateElement;
44 </pre>
45  - for all the items that do not starts with an underscore the getter will look into the public array and return the chosen item, false is returned if the item is not found.
46  '''php
47  <pre>
48  echo $config->myPublicElement;
49  </pre>
50 
51 3.The instance is destroyed:
52  * when the instance is destroyed the magic __destruct is invoked, if something has changed inside the config from the original file it will reserialize and save the new changes over to the json file.
